Frequently Asked Questions

What model does the system use?

It uses the Ultralytics YOLOv8 family (choose size per hardware: n/s/m/l/x). YOLOv8 offers a strong speed/accuracy trade-off and works well on both CPU and GPU.

References: YOLOv8 overview and model variants.

Which inputs are supported?

USB webcams (device indices like 0, 1) and RTSP network streams (e.g., from drones or IP cameras). OpenCV’s VideoCapture accepts either a device index or a stream/file URL.

How do I run it quickly?

From the repo root: python bear_launcher.py. This launches with defaults and opens your default camera. For RTSP, pass --source "rtsp://...". See the README for more examples and configuration.

What FPS should I expect?

Depends on model size, resolution, and hardware. Smaller YOLOv8 variants (e.g., yolov8n/yolov8s) are much faster; larger ones are more accurate but heavier.

Can I add more species or log detections?

Yes. The code is modular — you can add classes, change thresholds, or hook custom logging/alerts.

Ethics & privacy?

Use responsibly: avoid disturbing wildlife and follow local regulations. The system is intended for conservation and safety use cases.
