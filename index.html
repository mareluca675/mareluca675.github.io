Real-time bear detection for drones & ground cams

A modular, edge-ready computer-vision pipeline that ingests USB webcams and RTSP streams, runs YOLOv8 inference, and provides clear detections for wildlife research and safety.

View on GitHub Get Started

Why this system

Real-time

Optimized pipeline with fast models, practical defaults, and optional GPU usage to keep latency low during field operations.

Modular inputs

Drop-in sources: local cameras via device index and network cameras via RTSP, plus video files for offline testing.

Extensible

Swap model sizes, adjust thresholds, log detections, or add species later. The codebase is intentionally simple to modify.

How it works

The system reads frames from a camera or RTSP stream, preprocesses them, runs YOLOv8 detection, applies non-max suppression, and outputs detection events and overlays.

Get started (10 seconds)

Instant run (uses the launcher) 
python bear_launcher.py

Opens your default camera and starts detection with sensible defaults.

Manual run (with options) 
# Use a different camera
python bear_launcher.py --source 1

# Use an RTSP drone stream
python bear_launcher.py --source "rtsp://192.168.1.100:8554/stream"

# Adjust confidence threshold
python bear_launcher.py --conf 0.3

# Use a different model size
python bear_launcher.py --model yolov8s.pt

See the README in the GitHub repo for full examples and configuration.

Tip: For RTSP/IP cameras, use the full RTSP URL provided by your camera/drone. OpenCV’s VideoCapture can take a device index (e.g., 0) or a stream URL.
